# C++ 即時網路流量監控與攻擊偵測系統：學習指南

### 第二部分：答案金鑰
1. 第一週任務： 
    重點在於環境配置與基礎功能，包含安裝 Npcap/libpcap 並實作列出網卡與基本抓包功能。 第二週任務：  進入協定解析層，實作以解析 Ethernet、IP、TCP 與 UDP 標頭，提取 IP 位址、埠口與標誌位。

2. 推薦驅動：  
    推薦使用 Npcap v1.3+ SDK。因為 Npcap 是目前 Windows 上最新且持續維護的版本，相較於已停止維護的 WinPcap，其效能更好且支援 Windows 10/11。

3. 定義與角色：  
    Ring Buffer 是一種資料結構，用於協調抓包執行緒與處理執行緒。它能暫存擷取到的原始封包，避免因後端分析速度較慢而導致封包丟失，並解決多執行緒間的資料競爭問題。

4. 偵測方式：  
    系統會解析 TCP 標頭中的 SYN 標誌位，並按來源 IP 計數。若某一來源 IP 每秒發送的 SYN 封包超過 100 個且回傳的 ACK 數量極低，則判定為 SYN Flood 攻擊。

5. 實作邏輯：  
    使用 std::set 記錄特定 IP 在設定時間內（如 5 秒）接觸過的不同目標埠口數量。若該 IP 觸及的埠口超過預設閾值（如 20 個），則判定為掃描行為。

6. 偵測原理：  
    系統監聽 ARP 回應封包，並在本地維護一張「IP 對應 MAC」的表格。若發現同一個 IP 對應到多個不同的 MAC 位址，即代表偵測到 ARP 欺騙。

7. 優點：  
    透過 #pragma pack(push, 1) 禁用對齊，可以讓結構體與內存中的封包格式直接映射。這樣無需逐位元複製數據，能大幅縮短解析時間（約每封包 100 奈秒）。

8. 區別：  
    靜態規則使用固定的閾值（如每秒固定 100 次）；動態基準則會學習過去一段時間（如 5 分鐘）的平均流量，若當前流量遠高於此基線（如超過平均值的 3 倍），則判定為異常。

9. 優化方式： 
    採用預分配（Pre-allocation）技術，在程式啟動時即分配好大塊內存，避免在高流量下頻繁進行 malloc 或 new 操作，從而減少記憶體碎片與系統開銷。

10. 進階功能：  
    它允許系統讀取外部的 .rules 檔案，支援動態新增偵測邏輯。這使系統能靈活地過濾特定的應用層內容（如 HTTP GET）或惡意軟體特徵，而不需要重新編譯主程式。

### 第四部分：核心術語彙整 (Glossary)
術語,定義

Npcap / libpcap,用於封包擷取的核心驅動程式庫。Npcap 適用於 Windows，libpcap 則適用於 Linux/macOS。

PcapPlusPlus,一個現代化的 C++ 開發框架，提供對 Npcap/libpcap 的封裝，支援解析 DNS、HTTP、SSL 等應用層協議。

Ring Buffer (環形緩衝),一種定長的緩衝區，數據首尾相接，用於多執行緒環境下封包數據的高效緩衝。

SYN Flood,一種 DDoS 攻擊，透過發送大量偽造的 SYN 封包使目標伺服器資源耗盡。

ARP Spoofing (ARP 欺騙),攻擊者發送虛假 ARP 回應，將其 MAC 位址與合法 IP 綁定，藉此進行中間人攻擊。

Port Scanning (埠口掃描),攻擊者對目標主機的多個埠口進行探測，以找出開放的服務作為攻擊突破點。

Snort Rule,一種標準化的網路入侵偵測規則格式，包含動作、協定、來源/目標 IP 與埠口，以及內容檢查。

BPF Filters,Berkeley Packet Filter，一種在擷取層過濾封包的機制，能減少傳遞給應用程式的無效數據。

位域結構 (Bit-field),C++ 中精確控制數據位元的結構設計，常用於封裝與解析緊湊的網路協定標頭。

Lock-Free (無鎖結構),使用原子操作 (Atomic Operations) 而非傳統鎖機制來處理數據，以提高多執行緒併發效能。

Scapy,一款強大的 Python 工具，用於構建、發送封包，本專案中用於模擬各類攻擊進行測試。

CAS (Compare and Swap),一種 CPU 原子操作，用於在無鎖編程中安全地更新共享變數，防止資料競爭。
